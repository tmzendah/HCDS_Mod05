# Reproducibility

All 12 training runs (4 configurations × 3 seeds) can be reproduced using the instructions below. Results were produced on an NVIDIA A100-SXM4-80GB GPU via the Cambridge Service for Data Driven Discovery (CSD3).

---

## 1. Environment setup

### Option A — Conda (recommended)

```bash
conda env create -f environment.yml
conda activate knee_oa
```

### Option B — pip

```bash
python -m venv env
source env/bin/activate
pip install -r requirements.txt
```

Tested on: Python 3.11.15, PyTorch 2.7.1, CUDA 11.8, Linux x86_64.

---

## 2. Dataset download

**Kaggle Knee Osteoarthritis Dataset with Severity Grading** — CC BY 4.0
8,260 anteroposterior knee radiographs, KL grades 0–4, pre-split into `train/`, `val/`, `test/`.

```bash
pip install kaggle
# Place kaggle.json in ~/.kaggle/kaggle.json
kaggle datasets download -d shashwatwork/knee-osteoarthritis-dataset-with-severity
unzip knee-osteoarthritis-dataset-with-severity.zip -d data/
```

Expected structure after unzip:

```
data/
├── train/0/  train/1/  train/2/  train/3/  train/4/
├── val/0/    val/1/    val/2/    val/3/    val/4/
└── test/0/   test/1/   test/2/   test/3/   test/4/
```

---

## 3. Running experiments

### Submit all 12 jobs to CSD3 (recommended)

```bash
bash slurm/submit_all.sh
```

### Submit individual configurations

```bash
bash slurm/train_resnet50_ce_seed42.sh
bash slurm/train_resnet50_coral_seed42.sh
bash slurm/train_efficientnet_ce_seed42.sh
bash slurm/train_efficientnet_coral_seed42.sh
```

Each script submits one run (one config, one seed) as a separate SLURM job.

---

## 4. Training a single run manually

```bash
python src/train.py \
    --arch resnet50 \
    --loss ce \
    --seed 42 \
    --data_dir /rds/user/tm922/hpc-work/data/knee_oa \
    --output_dir results
```

---

## 5. Evaluation

```bash
python src/evaluate.py \
    --data_dir /rds/user/tm922/hpc-work/data/knee_oa \
    --results_dir results
```

Outputs per-seed metrics to `results/metrics/`. The aggregated summary across seeds is written to `results/metrics/summary.csv`.

---

## 6. Grad-CAM explainability

```bash
python src/gradcam_audit.py \
    --data_dir /rds/user/tm922/hpc-work/data/knee_oa \
    --results_dir results \
    --output_dir results/gradcam
```

Heatmaps for shared misclassification cases are saved to `results/gradcam/shared_misclassified/`.

---

## 7. Figures

Figures used in the report are generated by running:

```bash
python notebooks/results_analysis.py \
    --results_dir results \
    --output_dir results
```

Outputs are saved to `results/`.

---

## 8. Random seed control

Seeds 42, 123, and 456 are passed via `--seed` and applied to Python, NumPy, PyTorch, and CUDA random number generators at the start of each training run. Deterministic CUDA operations are enabled where available. Results may vary marginally across GPU hardware.

## 9. Expected results (test set, mean ± SD across 3 seeds)

| Configuration | QWK | KL1 Recall |
|---|---|---|
| ResNet50 + CE | 0.782 ± 0.003 | 0.127 ± 0.014 |
| ResNet50 + CORAL | 0.800 ± 0.005 | 0.341 ± 0.024 |
| EfficientNet-B0 + CE | 0.787 ± 0.002 | 0.190 ± 0.020 |
| EfficientNet-B0 + CORAL | 0.809 ± 0.009 | 0.354 ± 0.011 |

Full per-seed results are in `results/metrics/summary.csv`.
